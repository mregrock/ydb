# Обзор конфигурации

Для запуска узла {{ ydb-short-name }}, изменения параметров и добавления новых узлов в кластер требуется конфигурация. Начиная с версии v25.1 {{ ydb-short-name }} поддерживает механизм управления кластером, называемый [распределённой конфигурацией](../../concepts/glossary.md#distributed-configuration).

{% note info %}

Если ваш кластер был развернут до версии v25.1, то для перехода на автоматическую конфигурацию вы можете обратиться к соответствующей [инструкции](../deployment-options/manual/migration/migration-to-autoconfiguration.md).

{% endnote %}

Конфигурация представляет собой файл `config.yaml`, хранимый на узлах кластера и содержащий все его настройки.

Параметры конфигурации можно разделить на два типа, по способу их доставки и распространения на узлах кластера: [статические](#static-parameters) и [динамические](#dynamic-parameters). Подробнее параметры конфигурации описаны на странице [{#T}](../../reference/configuration/index.md).


## Статические параметры {#static-parameters}

Статические параметры применяются для запуска и настройки [статических узлов](../../concepts/glossary.md#static-node) кластера с помощью механизма [распределённой конфигурации](../../concepts/glossary.md#distributed-configuration). Параметры содержат базовые характеристики узла хранения, такие как:

- информация о хостах и дисках;
- параметры безопасности;
- настройки [акторной системы](../../concepts/glossary.md#actor-system);
- другие критические настройки кластера.

## Динамические параметры {#dynamic-parameters}

Динамические параметры необходимы для запуска [динамических узлов](../../concepts/glossary.md#dynamic-node) кластера. Конфигурация надёжно сохраняется в [таблетке](../../concepts/glossary.md#tablet) Console и с помощью внутренних механизмов таблетки доставляется до узлов кластера.

Для более сложных сценариев применения динамических параметров можно использовать механизмы [селекторов](./dynamic-config-selectors.md) и [временной конфигурации](./dynamic-config-volatile-config.md).

## Версионирование конфигурации

Данный механизм позволяет избежать конкурентной модификации конфигурации и сделать её обновление идемпотентным. При получении запроса на модификацию конфигурации сервер сравнивает версию полученной модификации с сохранённой:

- Если версия меньше на единицу, то конфигурации сравниваются — если они одинаковы, значит пользователь пытается загрузить конфигурацию повторно, пользователь получает ОК, а конфигурация на кластере не обновляется.
- Если версия равна текущей на кластере, то конфигурация заменяется на новую, при этом поле версии увеличивается на единицу.
- Во всех остальных случаях пользователь получает ошибку.
